# Data Model: Receipt Search and Cancel

**Feature**: 085-receipt-search-cancel
**Date**: 2026-05-30

The `payments` table already contains all fields required for this feature. No schema changes.

### Indexes (existing)

| Index Name | Columns | Purpose |
|------------|---------|---------|
| `idx_payments_tenant_receipt` | `tenant_id, receipt_number` | Receipt number search/filter |
| `idx_payments_receipt_number` | `tenant_id, receipt_number` | Non-unique lookup for grouped receipts |
| `idx_payments_campaign_void` | `tenant_id, fee_campaign_id, voided_at` | Campaign payment void filtering |

## State Transitions

```
[ACTIVE] --(cancel with reason)--> [VOIDED]
    ^                                    |
    |                                    |
    +-----(un-void is OUT OF SCOPE v1)---+
```

**Active** → **Voided**:
- Trigger: `POST /api/payments/:id/void` with valid reason
- Effect: `voided_at = NOW()`, `void_reason = {reason}`, `voided_by = {userId}`
- Side effects: Re-run `LedgerService::allocatePaymentToCharges()` for the student
- Irreversible for v1

## Relationships

- **Payment → Student**: `payments.student_id` → `students.id` (many-to-one)
- **Payment → Tenant**: `payments.tenant_id` → `tenants.id` (many-to-one)
- **Payment → Receipt Group**: `payments.payment_group_id` links sibling rows sharing the same `receipt_number`

## Validation Rules

- `receipt_number` must be 25 characters or less
- `void_reason` is required (non-empty string, max 500 characters) on void action
- `voided_at` is set server-side; never accepted from client
- Only `admin` or `bursar` roles may trigger void state transition
- Double-void is prohibited (409 Conflict if `voided_at` is already set)
- Grouped payments: all rows sharing `payment_group_id` must be voided together (atomic)

## Query Patterns

1. **Receipt search**: `WHERE tenant_id = ? AND receipt_number LIKE '%{term}%'` — uses `idx_payments_tenant_receipt`
2. **Active payments only**: `WHERE tenant_id = ? AND voided_at IS NULL` — used for financial totals
3. **Voided payments filter**: `WHERE tenant_id = ? AND voided_at IS NOT NULL` — used for audit views
4. **Ledger balance**: Payment subqueries must include `AND voided_at IS NULL` to exclude voided amounts
