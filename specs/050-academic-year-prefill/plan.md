# Implementation Plan: Academic Year Auto-Prefill for Migration Form

**Branch**: `050-academic-year-prefill` | **Date**: 2026-04-28 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/050-academic-year-prefill/spec.md`

## Summary

Enhance the year-end class migration form (`YearEndMigrationPanel`) so that the "From Academic Year" and "To Academic Year" fields are automatically prefilled by the server on form load — no manual typing required. The server derives the current active academic year from the tenant's term calendar (or falls back to `settings.academicYear`), computes the next year arithmetically, and returns both values plus a list of all known years via a new `GET /api/class-migration/year-prefill` endpoint. The frontend replaces the existing free-text `<Input>` fields with `<Select>` dropdowns; options are badged ("Current Active", "Next", "To Be Created"). No schema migrations are required.

## Technical Context

**Language/Version**: PHP 8.1 (backend) · TypeScript / React 18 (frontend)  
**Primary Dependencies**: CodeIgniter 4 · MySQL · Vite · TailwindCSS · shadcn/ui · TanStack React Query · React Hook Form · Zod  
**Storage**: MySQL — reads `tenants.academic_calendar` (JSON) and queries `class_instances.academic_year` (VARCHAR). Zero writes for the prefill endpoint itself.  
**Testing**: CodeIgniter 4 FeatureTestTrait + DatabaseTestTrait (backend integration tests in `backend/tests/`)  
**Target Platform**: Linux server (backend) · Browser SPA (frontend)  
**Project Type**: Web service (REST API) + React SPA  
**Performance Goals**: Prefill endpoint responds in ≤ 300 ms (single `tenants` row read + one `DISTINCT` query on `class_instances`)  
**Constraints**: Zero schema migrations; must not alter existing `preview`/`run` API contracts; tenant isolation required on all backend queries  
**Scale/Scope**: Per-tenant operation; up to 20 academic year strings; minimal data volume
